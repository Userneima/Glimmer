# Hooks Module Rules

它只回答一个问题：改业务 hook 时，哪些状态流不能被打断。

## 接手入口

- `useDiaries.ts` 是日记创建、更新、删除、导入、当前选中状态的主入口。
- `useTasks.ts` 是任务聚合、提醒事项、本地任务、任务文档相关行为的重要入口。
- `useDiaryTagActions.ts` 负责标签选择、重命名、合并、删除，不要把这类批量标签逻辑塞回布局组件。
- `useAutoDiaryAnalysis.ts` / `useAutoDiaryTags.ts` 是后台增强，不应打扰写作。
- `useReviewDataSync.ts` 只负责 AI 回看数据本地/云端合并，失败时必须静默降级。
- `useLocalBackup.ts` 负责桌面本地备份触发，不应和页面布局耦合。
- `useLongTermIdeas.ts` 管长期想法，不要与普通日记逻辑混淆。

## 边界

- 本地写入优先，云同步失败不能阻塞日记编辑。
- 自动 AI 分析和自动标签必须低打扰，不能强行生成低质量长期想法或低复用标签。
- 创建日记要尊重模板日记机制；日历按指定日期创建时不能回退到今天。
- hook 不应直接包含大块展示 UI。

## 验证

- 改 `useDiaries.ts` 后复核新建、编辑、删除、刷新保留、云同步降级。
- 改任务 hook 后复核提醒事项刷新、本地任务、任务文档绑定。
- 改自动分析/标签后复核不会重复生成、不会污染旧标签。
