# CURRENT_STATE

它只回答一个问题：**Glimmer 现在已经做到哪一步，哪些能力已存在，哪些区域接手前要先复核。**

## 当前项目快照
Glimmer 现在已经不是早期 demo，而是一个以本地 `localStorage` 为主、可选接入 Supabase 云同步的个人写作产品。主界面围绕“左侧组织与视图切换 + 中间日记正文编辑 + 若干增强面板/弹窗”组织。当前核心状态是：写作、组织、任务、导出、模板、长期想法、可选账号与 AI 配置都已接入同一个产品里。

## 已稳定存在的能力
### 富文本与正文编辑
- 使用 TipTap 作为编辑核心，支持标题、列表、任务列表、引用、图片、表格、链接、分隔线等常见正文能力。
- 表格不是静态装饰，已经支持列宽调整、行列增删、合并拆分、水平/垂直对齐，以及表格内选择范围优化。
- 列表支持真实层级缩进，不再只是插空格。

### 组织与回看
- 左侧已具备文件夹、标签、日历、任务四个主视图入口。
- 日历视图已经改为“当天优先”，月历通过弹窗查看，并能预览选中日期的日记和任务。
- 标签系统不仅是筛选，还支持颜色、重命名、合并。

### 任务与长期想法
- 任务支持长期任务和时间范围任务，能按日期回看。
- 长期想法不是独立产品，而是从日记正文中抽取、回看、回跳原文的位置型增强能力。

### 导出与模板
- 导出已支持 Markdown、HTML、JSON、DOCX、PDF。
- 导出不仅支持“当前日记”，也支持“全部日记”，并能选择合并单文件或拆分多文件。
- 新建日记不再是空白文档，而是复制一篇专门的模板日记当前状态。

### 同步、认证与 AI 配置
- 本地优先仍然成立；即使 Supabase 不可用，本地仍应工作。
- 账号系统基于 Supabase Auth，当前方向是内部账号优先，而不是公开注册产品。
- AI 配置不再只是本地状态，已经有“与账号绑定”的云端配置表读写。

## 特殊机制与系统实体
- `LONG_TERM_MASTER_ID`：长期想法主日记，是系统实体，不是普通用户日记。
- `TEMPLATE_DIARY_ID`：模板日记，新建日记时复制它的当前内容和标题规则。
- `Diary.content`：正文真相源。富文本、表格、任务列表样式都围绕这份 HTML 内容保存和导出。
- localStorage 使用当前用户 ID 做 key 前缀；登录态切换会影响读取哪一组本地数据。

## 最近变化较多、接手前应先复核的区域
- 表格编辑：最近围绕列宽、对齐、选择、最小宽度和交互细节做过连续调整。
- 模板日记：从“设置里写模板文本”改成了“单独模板日记复制”方案。
- 日历视图：最近重排为当天优先，月历变成弹窗预览，不应再按旧常驻月历理解。
- 长期想法窗口：经历过多轮 UI 收缩和产品风格回归，窗口比例和风格边界是敏感区。
- 认证与 AI 配置：README 和旧文档中仍保留过时说法，代码才是准确信息源。

## 当前事实源优先级
如果另一位 AI 要快速建立正确认知，优先看这些代码入口：
- `src/types/index.ts`：特殊实体、核心类型、系统 ID。
- `src/hooks/useDiaries.ts`：日记创建、模板复制、系统日记补齐、本地与云端融合。
- `src/utils/storage.ts`：本地优先与按用户隔离的持久化结构。
- `src/utils/cloud.ts`：云端映射、Supabase 兼容与 fallback。
- `src/context/AuthContext.tsx`：认证流、内部账号限制、AI 设置同步。
